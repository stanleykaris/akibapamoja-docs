# Send OTP

Sends a one-time password (OTP) to the user's registered phone number or email address for verification.

## Endpoint

```
POST /users/otp/send/
```

## Request Body

| Field    | Type   | Required | Description                        |
|----------|--------|----------|------------------------------------|
| `identifier` | string | Yes   | Phone number or email to send OTP to |
| `channel` | string | No     | Delivery channel: `sms` or `email`. Defaults to `sms` |

### Example

```json
{
  "identifier": "+254712345678",
  "channel": "sms"
}
```

```bash
curl -X POST https://akibapamoja-backend.onrender.com/api/users/otp/send/ \
  -H "Content-Type: application/json" \
  -d '{"identifier": "+254712345678", "channel": "sms"}'
```

## Response

### 200 OK

OTP sent successfully.

```json
{
  "message": "OTP has been sent successfully.",
  "expires_in": 300
}
```

### 400 Bad Request

Invalid identifier or rate limit exceeded.

```json
{
  "detail": "Too many OTP requests. Please try again later."
}
```

## Notes

- OTPs expire after **5 minutes** (300 seconds).
- Rate limiting applies to prevent abuse.
- After receiving the OTP, use the [Verify OTP](/docs/api/users_otp_verify) endpoint to complete verification.
