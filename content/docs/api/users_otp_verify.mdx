# Verify OTP

Verifies a one-time password (OTP) that was previously sent via the [Send OTP](/docs/api/users_otp_send) endpoint.

## Endpoint

```
POST /users/otp/verify/
```

## Request Body

| Field    | Type   | Required | Description                      |
|----------|--------|----------|----------------------------------|
| `identifier` | string | Yes   | The phone number or email the OTP was sent to |
| `otp_code` | string | Yes    | The OTP code received by the user |

### Example

```json
{
  "identifier": "+254712345678",
  "otp_code": "123456"
}
```

```bash
curl -X POST https://akibapamoja-backend.onrender.com/api/users/otp/verify/ \
  -H "Content-Type: application/json" \
  -d '{"identifier": "+254712345678", "otp_code": "123456"}'
```

## Response

### 200 OK

OTP verified successfully. Returns user authentication tokens.

```json
{
  "message": "OTP verified successfully.",
  "access": "eyJhbGciOiJIUzI1NiIs...",
  "refresh": "eyJhbGciOiJIUzI1NiIs..."
}
```

### 400 Bad Request

Invalid or expired OTP.

```json
{
  "detail": "Invalid or expired OTP."
}
```

### 400 Too Many Attempts

Maximum verification attempts exceeded.

```json
{
  "detail": "Too many failed attempts. Please request a new OTP."
}
```

## Notes

- OTPs expire after **5 minutes**.
- Maximum **3 failed attempts** per OTP before it is invalidated.
- On success, returns the same token pair as [Obtain Token Pair](/docs/api/users_auth_obtain-pair_create).
