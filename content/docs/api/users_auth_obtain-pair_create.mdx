# Obtain Token Pair

Retrieves a new access and refresh token pair using user credentials. This is the primary authentication endpoint.

## Endpoint

```
POST /users/auth/obtain-pair/
```

## Request Body

| Field    | Type   | Required | Description                    |
|----------|--------|----------|--------------------------------|
| `email`  | string | Yes      | User's registered email address |
| `password` | string | Yes    | User's password                |

### Example

```json
{
  "email": "user@example.com",
  "password": "securepassword123"
}
```

```bash
curl -X POST https://akibapamoja-backend.onrender.com/api/users/auth/obtain-pair/ \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com", "password": "securepassword123"}'
```

## Response

### 200 OK

Returns both tokens on successful authentication.

```json
{
  "access": "eyJhbGciOiJIUzI1NiIs...",
  "refresh": "eyJhbGciOiJIUzI1NiIs..."
}
```

### 400 Bad Request

Invalid credentials or missing fields.

```json
{
  "detail": "No active account found with the given credentials"
}
```

## Token Usage

- **Access Token**: Include in `Authorization: Bearer <token>` header for authenticated API calls. Short-lived.
- **Refresh Token**: Use with the [Refresh Token](/docs/api/users_auth_refresh_create) endpoint to get a new access token without re-authenticating.
